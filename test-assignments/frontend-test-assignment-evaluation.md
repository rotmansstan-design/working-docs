# Критерии проверки: Frontend Test Assignment (React + Next.js)

## Принцип оценки

Формат оценки: **Core (70 баллов) + Optional (30 баллов)**.  
Сильный core важнее количества фич.

Итог: **100 баллов**.

## Core — 70 баллов

### 1) Корректность пользовательского флоу — 25 баллов

Проверяем:
- валидная работа формы на happy-path;
- корректная обработка `409` и сетевых ошибок;
- предсказуемые UI-состояния.

### 2) Устойчивость и идемпотентность UI — 20 баллов

Проверяем:
- защита от двойного submit;
- корректное использование `idempotency_key`;
- отсутствие дублей в клиентском поведении.

### 3) Архитектура и читаемость — 15 баллов

Проверяем:
- разделение UI/state/API;
- понятная структура кода;
- отсутствие «бог-компонента».

### 4) Безопасность и тесты — 10 баллов

Проверяем:
- отсутствие явно небезопасных паттернов;
- минимальный набор тестов на критичный флоу.

## Optional — 30 баллов

Проверяем:
- восстановление состояния после reload;
- дополнительный E2E/интеграционные кейсы;
- осмысленные оптимизации производительности;
- качество объяснения trade-off'ов.

## Красные флаги

- двойной submit приводит к дублям;
- критичные ошибки не обработаны для пользователя;
- тестов на основной флоу нет;
- явно небезопасные решения без объяснений.

## Интерпретация итогового балла

- **85-100**: Strong hire
- **70-84**: Hire
- **55-69**: Borderline
- **<55**: No hire
