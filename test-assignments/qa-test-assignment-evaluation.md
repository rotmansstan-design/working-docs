# Критерии проверки: QA Automation Test Assignment

## Принцип оценки

Формат оценки: **Core (70 баллов) + Optional (30 баллов)**.  
Оценивается качество отбора рисков и практичность тестовой стратегии.

Итог: **100 баллов**.

## Core — 70 баллов

### 1) Приоритизация рисков и тест-стратегия — 25 баллов

Проверяем:
- понятно, почему выбраны именно эти кейсы;
- фокус на финансово-критичных рисках;
- ясные границы покрытия.

### 2) Качество API-автотестов — 25 баллов

Проверяем:
- есть граничные сценарии;
- проверена идемпотентность;
- тесты проверяют поведение, а не только статус-коды.

### 3) Качество E2E-базы — 15 баллов

Проверяем:
- критичный пользовательский путь покрыт;
- есть минимум один негативный сценарий;
- тесты воспроизводимы.

### 4) Артефакты и воспроизводимость — 5 баллов

Проверяем:
- README и TEST_STRATEGY понятны;
- тесты запускаются по инструкции.

## Optional — 30 баллов

Проверяем:
- дополнительная защита от double-submit;
- smoke-нагрузка/устойчивость;
- CI-интеграция;
- зрелость анализа trade-off'ов.

## Красные флаги

- нет тестов на идемпотентность;
- отсутствуют негативные API-кейсы;
- E2E не покрывает базовый пользовательский флоу;
- запуск тестов нельзя воспроизвести.

## Интерпретация итогового балла

- **85-100**: Strong hire
- **70-84**: Hire
- **55-69**: Borderline
- **<55**: No hire
